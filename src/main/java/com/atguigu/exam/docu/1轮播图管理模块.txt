        2.2.没有排序，根据banners表中filed sortOrder排序 （越小越优先）
                解决：
                 LambdaQueryWrapper<Banner> queryWrapper = new LambdaQueryWrapper<>();
                    queryWrapper.orderByAsc((banner) -> {
                        return banner.getSortOrder();
                 });
        2.3.返回多余字段
            我们不需要的字段，但是返回了  例如 updataTime idDeleted
            后端返回给前端的json是  实体类 -转换- json  ，实体类有就会转换
            忽略不需要的字段：通过jackson依赖包 ，
            controller -》 handler()方法 -》 java实体类  -转>  json   -发送》前端
                                                 是springmvc的handlerAdapter组件 掉用外部jackson完成 实体类转json
            所以：用jackson提供的注解，来完成忽略实体类字段     -》@JsonIgnore注解
            @JsonIgnore：加上后，不接收前端发送的被忽略的字段， 传给前端的json也 会忽略被注释的字段

        2.4.时间格式错乱，时区有问题
            jackson外部依赖，也是把实体类的时间转成 json的时间 要用的包
            解决： 用jackson包的@Jsonformt()
                方案1：局部@JsonFormat(pattern = "yyyy-HH-dd HH:mm:ss" , timezone = "GMT+8"  )
                2  : 全局配置
                spring：
                  jackson:
                    date-format: yyyy-MM-dd HH:mm:ss
                    time-zone: GMT+8

        2.5.被逻辑删除的字段，没有被排除
            1.全局：
                mybatis-plus:
                  global-config:
                    db-config:
                      logic-delete-field: isDeleted  # isDeleted是实体类的字段
            2.局部
                @JsonIgnore
                @Schema(description = "逻辑删除")
                @TableField("is_deleted")
                @TableLogic     //mybatis-plus的局部逻辑删除
                private Byte isDeleted;
            3.条件
                LambdaQueryWrapper<Banner> queryWrapper = new LambdaQueryWrapper<>();
                        queryWrapper.orderByAsc(Banner::getSortOrder);
                //        queryWrapper.eq(Banner::getIsDeleted , 0 );

                //        1. 查询所有轮播图数据集合
                //        2. 封装到"entity"类中
                        List<Banner> bannerList = bannerService.list(queryWrapper);

## 轮播图模块总结

本模块是一个典型的单表CRUD操作，主要技术要点如下：

### 1. MinIO集成
- 引入了MinIO外部依赖，实现分布式文件存储
- 通过配置MinIO客户端完成文件上传功能

### 2. 逻辑删除机制
- 删除操作采用逻辑删除而非物理删除
- 可通过两种方式实现：
  - 全局配置：在application.yml中统一配置逻辑删除字段
  - 局部注解：使用MyBatis-Plus提供的 `@TableLogic` 注解

### 3. 时间字段自动维护
- `create_time`（创建时间）和 `update_time`（更新时间）字段由MySQL自动维护
- 无需在业务代码中手动设置，减少代码冗余

### 4. 后续优化方向
- 将日志记录功能提取为独立的AOP切面类（BannerControllerAspect）
- 通过面向切面编程实现日志的统一管理，提高代码可维护性

### 5. MultipartFile接口学习
- `MultipartFile` 是Spring框架提供的文件上传接口
- 该接口有两个实现类：
  - 生产级实现：官方已将其构造方法私有化，不推荐直接使用
  - 测试级实现：仅用于单元测试场景
- **核心思想**：我们只需关注接口定义的方法，无需关心具体实现
- 框架底层会自动完成文件数据的注入和封装，开发者只需调用接口方法即可
